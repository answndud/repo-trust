# JSON Report Reference

RepoTrust JSON output is the machine-readable report contract for CI,
dashboards, and other automation. The current contract version is
`schema_version: "1.2"`.

Use this reference when parsing output from:

```bash
repo-trust json <target>
repo-trust gate <target>
repotrust scan <target> --format json
```

Terminal dashboards and status messages are written to stderr. JSON report
content is written to stdout or to the `--output` file, so redirected output
remains valid JSON.

## Top-Level Object

| Key | Type | Description |
| --- | --- | --- |
| `schema_version` | string | JSON report contract version. Current value: `"1.2"`. |
| `target` | object | Parsed target input and repository location fields. |
| `detected_files` | object | Repository file signals RepoTrust found or could not find. |
| `findings` | array | Ordered list of finding objects. Empty when no enabled checks found issues. |
| `score` | object | Category scores, total score, grade, and risk label. |
| `assessment` | object | Overall verdict plus purpose-specific profiles. |
| `generated_at` | string | UTC ISO-8601 timestamp generated by the local run. |

## `target`

| Key | Type | Description |
| --- | --- | --- |
| `raw` | string | Original CLI target string. |
| `kind` | string | `local` or `github`. |
| `path` | string or null | Local filesystem path for local targets. |
| `host` | string or null | GitHub host for GitHub targets. |
| `owner` | string or null | GitHub owner or organization. |
| `repo` | string or null | GitHub repository name. |
| `ref` | string or null | Parsed branch, tag, or commit ref for GitHub tree/blob URLs. |
| `subpath` | string or null | Parsed tree/blob subpath. Remote scans still assess repository-root signals and report a subpath limitation finding when this is present. |

## `detected_files`

| Key | Type | Description |
| --- | --- | --- |
| `readme` | string or null | README path when found. |
| `license` | string or null | License file path when found. |
| `security` | string or null | Root `SECURITY.md` or `.github/SECURITY.md` path when found. |
| `ci_workflows` | array of strings | CI workflow paths. |
| `dependency_manifests` | array of strings | Package manifest paths. |
| `lockfiles` | array of strings | Lockfile paths. |
| `dependabot` | string or null | Dependabot config path when found. |

Remote scans may set a signal to `null` or an empty array because the file is
missing, or because that specific endpoint could not be checked. Use
`findings` and `assessment.coverage` to distinguish missing evidence from
partial scan coverage.

## `findings[]`

| Key | Type | Description |
| --- | --- | --- |
| `id` | string | Stable finding ID for policy exceptions and automation. |
| `category` | string | One of `readme_quality`, `install_safety`, `security_posture`, `project_hygiene`, or `target`. |
| `severity` | string | One of `info`, `low`, `medium`, or `high`. |
| `message` | string | Human-readable finding summary. |
| `evidence` | string | Short evidence string, usually a path, command, or API status. |
| `recommendation` | string | Suggested remediation or review action. |

Finding IDs are documented separately in
[finding-reference.md](finding-reference.md). For automation, key policy logic
should use `id`, `category`, `severity`, and assessment verdicts rather than
human-readable `message` or `recommendation` text.

## `score`

| Key | Type | Description |
| --- | --- | --- |
| `categories` | object | Category score map. Current keys are `readme_quality`, `install_safety`, `security_posture`, and `project_hygiene`. |
| `total` | integer | Weighted total score after score caps are applied. |
| `max_score` | integer | Maximum possible score. Current value: `100`. |
| `grade` | string | Letter grade derived from `total`. |
| `risk_label` | string | Human-readable risk band derived from `total`. |

Score caps can reduce `total` when scan coverage is incomplete or the target is
unsupported. Category scores still show deductions from enabled findings.

## `assessment`

| Key | Type | Description |
| --- | --- | --- |
| `verdict` | string | Overall verdict for installing, depending on, or delegating the repository. |
| `confidence` | string | `high`, `medium`, or `low`. |
| `coverage` | string | `full`, `partial`, `metadata_only`, or `failed`. |
| `summary` | string | Short human-readable summary. |
| `reasons` | array of strings | Main reasons behind the verdict. |
| `next_actions` | array of strings | Suggested next actions. |
| `profiles` | object | Purpose-specific verdict map. |

Verdict values, from most restrictive to least restrictive:

- `do_not_install_before_review`
- `insufficient_evidence`
- `usable_after_review`
- `usable_by_current_checks`

## `assessment.profiles`

Current profile keys:

- `install`: whether to run the documented install commands.
- `dependency`: whether to add the repository/package as a dependency.
- `agent_delegate`: whether to let an AI agent install or operate on the repository.

Each profile object has this shape:

| Key | Type | Description |
| --- | --- | --- |
| `verdict` | string | Profile-specific verdict using the same verdict IDs as `assessment.verdict`. |
| `summary` | string | Profile-specific summary. |
| `reasons` | array of strings | Profile-specific reasons. |
| `next_actions` | array of strings | Profile-specific next actions. |
| `priority_finding_ids` | array of strings | Up to three finding IDs most relevant to this profile. |

## Compatibility Notes

- `schema_version` remains `"1.2"` while changes only add new stable finding
  IDs, renderer behavior, docs, examples, or wording.
- Adding, removing, or renaming top-level keys requires an intentional schema
  review and likely a version change.
- Adding, removing, or renaming keys inside `target`, `detected_files`,
  `findings[]`, `score`, `assessment`, or `assessment.profiles.*` requires an
  intentional schema review.
- New finding IDs may be added without changing `schema_version`.
- Human-readable text fields may change without a schema version change.
- `repo-trust gate` always writes JSON first, then returns exit code `1` when
  score or profile policy fails.

## `jq` Examples

Read the overall verdict:

```bash
jq -r '.assessment.verdict' repotrust-report.json
```

Read the AI agent delegation verdict:

```bash
jq -r '.assessment.profiles.agent_delegate.verdict' repotrust-report.json
```

List medium and high severity finding IDs:

```bash
jq -r '.findings[] | select(.severity == "medium" or .severity == "high") | .id' repotrust-report.json
```

Check whether the report has full coverage:

```bash
jq -e '.schema_version == "1.2" and .assessment.coverage == "full"' repotrust-report.json
```

## Python Example

```python
import json
from pathlib import Path

report = json.loads(Path("repotrust-report.json").read_text(encoding="utf-8"))

if report["schema_version"] != "1.2":
    raise SystemExit(f"Unsupported RepoTrust schema: {report['schema_version']}")

blocking = {
    finding["id"]
    for finding in report["findings"]
    if finding["severity"] in {"medium", "high"}
}

agent_verdict = report["assessment"]["profiles"]["agent_delegate"]["verdict"]
if blocking or agent_verdict != "usable_by_current_checks":
    raise SystemExit("Review required before agent delegation")
```
